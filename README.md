## üìÅ Asset Management

### üñºÔ∏è Image Assets

WmaskEX recursively searches for image files with the following extensions (case-insensitive):
- **Supported formats**: `png`, `jpg`, `jpeg`, `bmp`, `ico`, `tiff`, `exif`, `wmf`, `emf`

### üé≠ Spine Animation Assets

For Spine animations to be recognized, a folder must contain:
- ‚úÖ `.atlas` file (texture atlas)
- ‚úÖ `.json` or `.skel` file (skeleton data) 
- ‚úÖ `.wmaskex.json` or `.wmaskex.autogen.json` file (WmaskEX configuration)

**All files must share the same name as their containing folder.**

#### Spine Configuration Example

`.wmaskex.json` or `.wmaskex.autogen.json` (auto-generated by WmaskEXSpineConfigAutoGenerator):

```json
{
    "version": "3.8", 
    "bounds": [-100, -10, 220, 450], 
    "pma": true
}
```

**Bounds Format**: `[x, y, width, height]`
- `width`, `height`: Bounding box dimensions
- `x`, `y`: Bottom-left corner coordinates
- Auto-generation uses the maximum bounding box across all animations

## üõ†Ô∏è Installation & Setup

### System Requirements
- Windows 7/8/10/11 (64-bit)
- Visual C++ Redistributable
- OpenGL support

### Quick Start
1. Download the latest release
2. Extract to your preferred directory
3. Run `WmaskEX.exe`
4. Configure your overlays through the GUI
5. Enjoy your enhanced desktop experience!

### Building from Source

```powershell
# Install dependencies with vcpkg
vcpkg install nlohmann-json glbinding

# Clone and build
git clone https://github.com/wang606/WmaskEX.git
cd WmaskEX
mkdir build && cd build
cmake ..
cmake --build . --config Release
```
